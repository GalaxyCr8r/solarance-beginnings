# Galaxy Creator (privileged admin client)

A separate, admin/moderator-only client for seeding and editing the galaxy at
runtime — **without** re-publishing the SpacetimeDB module. Implements issue
[#34](https://github.com/GalaxyCr8r/solarance-beginnings/issues/34).

It is deliberately utilitarian (function over form). It mirrors the
macroquad + egui + `spacetimedb-sdk` stack of [`../client`](../client) so the
star-system rendering code can eventually be lifted over to preview new systems
and sectors before committing them.

## What it can do (M4 scope)

1. **Create sector** — pick a star system + controlling faction, set name,
   security level, ambient parameters (sunlight / anomalous / nebula / rare-ore),
   and star-system position.
2. **Place station** — into a sector, choosing size and owner faction. Two
   modes:
   - *Construction site* — an under-construction station players contribute
     resources to (with an editable requirements list).
   - *Finished station* — an operational station placed directly, optionally
     fitted with modules (trading port, refineries, solar array, manufacturing).
3. **Connect sectors** — link two existing sectors with a bidirectional jumpgate.
4. **Add module to station** — fit a module (trading port, refineries, solar
   array, manufacturing) onto an *existing* station. Construction sites complete
   with **no** modules, so this is how a finished site gets something to do.
   The per-size module cap is enforced server-side.

Each action calls a `try_server_only`-guarded `admin_*` reducer on the server.
The right-hand panel lists current galaxy state and an activity log that surfaces
the server's success/error for every call.

## Authentication

These reducers only run for the **module owner**, so the client must connect
with the owner's secret token. Paste it into the connection dialog. Get it with:

```bash
spacetime login show --token   # or: task show-token
```

The token's identity must be on the server's `try_server_only` allow-list
(`server/src/utility.rs`).

## Running

```bash
# from this directory
task generate   # regenerate bindings after server changes (spacetime generate)
task run        # cargo run

# or plain cargo
cargo run
```

In the connection dialog choose a server (`localhost`, `maincloud`, or a custom
host), confirm the database name (`solarance-beginnings`), paste the owner token,
and click **Connect**.

## Notes

- `src/server/bindings/` is generated by `spacetime generate` and is **not**
  committed (same convention as `../client`). Run `task generate` after pulling.
- This binary is never bundled into the player build.
